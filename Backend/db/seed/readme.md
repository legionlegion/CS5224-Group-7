# Seed Data

This folder stores exported Firebase Emulator state used as fixture data for local development and testing.

## How it works

When you run the emulator with:

```bash
firebase emulators:start --import=./seed --export-on-exit=./seed
```

- On **start**: emulator loads Firestore, Auth, and Storage data from this folder
- On **exit**: emulator saves the current state back here

## Committing seed data

Commit this folder (including subdirectories generated by `--export-on-exit`) so all team members share the same baseline test data.

## Resetting seed data

To start fresh, delete the contents of this folder (keep this readme) and re-run the emulator. Populate data manually via the Emulator UI at http://localhost:4000, then stop the emulator to export.
